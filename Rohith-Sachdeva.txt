    37. Walk me through Lambda's git flow. 

    37.  You need to fork/create your own version of the project repository.  Then add your TL as a collaborator.  Clone the repo by copying the web URL.  In your command line app, use the cd command to get to the directory where you want the repository saved. Then you need to use "git clone <link-to-repo>" to clone the repository to the directory you chose.  Now change directory into the newly created directory using cd <repo-name>.  Now you have to use the 'git checkout -b 'firstname-lastname'' command to create your own branch for the repo.  Work on the assignment.  Use "git add ." will add the repository changes.  Use "git commit -m "Your helpful commit message here"" to commit these changes, add a message that describes the changes.  Then you need to push these changes back to github from your local machine.  This is done via "git push -u origin branch-name".  After that you need to select your forked repository and switch to your newly pushed branch.  Create a pull request and choose your forked repository, not the master one.  Add any people to review this pull request.  Then those who have access can merge the pull request after they've reviewed it.  

Stretch Questions

1. What is the difference between an inline element and a block element?

1. A block element will naturally fill its parent's container and can have margins/paddings.  Such as <p> <div> <header> etc.  An inline element can be used in a text without causing it to drop to the next line, which block elements do naturally.  Inline elements include <a> <span> <b> etc.  

2. What happens when an element is positioned absolutely? 

2. It will be placed relative to the intial containing block via the values top, right, bottom, and left.
